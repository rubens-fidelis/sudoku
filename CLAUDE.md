# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

A Godot 4.6 Sudoku game targeting Android, iOS, Windows, macOS, and Web. Built entirely in GDScript with no third-party runtime dependencies beyond the bundled GUT test plugin.

## Running and Testing

**Run the game:** Open in Godot 4.6 editor and press F5.

**Run all tests (GUT):**
```sh
godot --path . -s addons/gut/gut_cmdln.gd
```

**Run a single test file:**
```sh
godot --path . -s addons/gut/gut_cmdln.gd -gtest=res://tests/test_sudoku.gd
godot --path . -s addons/gut/gut_cmdln.gd -gtest=res://tests/test_game_state.gd
```

Test files: `tests/test_sudoku.gd` (puzzle logic) and `tests/test_game_state.gd` (state, signals, pencil marks, win condition).

## Architecture

**Scene-per-screen with a global autoload as the single source of truth.**

```
main_menu.tscn → game.tscn → win_screen.tscn
```

Scene transitions use `get_tree().change_scene_to_file(...)`. Scenes never communicate directly — all shared state flows through the `GameState` autoload singleton.

### Key files

| File | Role |
|---|---|
| `scripts/game_state.gd` | Autoload singleton — owns `board`, `solution`, `clues`, `pencil_marks`, `difficulty`, `error_count`, `locale`; emits `board_changed`, `game_won`, `errors_changed`; manages settings persistence |
| `scripts/sudoku.gd` | Pure logic node (no UI): `generate(difficulty)`, `_solve()`, `is_valid()`, `_remove_cells()` (uniqueness-safe), `_count_solutions()` |
| `scripts/game.gd` | Game screen — builds 81-cell grid, handles input, calls `GameState` to mutate state; disconnects autoload signals in `_exit_tree()` |
| `scripts/cell.gd` | Individual cell panel — renders big number or 3×3 pencil marks; emits `cell_selected` |
| `scripts/number_picker.gd` | 1–9 + clear button row; emits `number_pressed` and `clear_pressed` |
| `scripts/main_menu.gd` | Main menu with difficulty buttons and language selector (OptionButton) |
| `scripts/win_screen.gd` | Win screen with play-again button |

### Board representation

Flat 81-element `Array` of ints. `pos = row * 9 + col`. Zero means empty.

### Difficulty removals

| Difficulty | Cells removed | Clues remaining |
|---|---|---|
| easy | 36 | 45 |
| medium | 46 | 35 |
| hard | 54 | 27 |

All generated puzzles have exactly one solution — `_remove_cells()` calls `_count_solutions()` and restores cells that would create ambiguity.

## Localization (i18n)

Translation files live in `locale/` as gettext PO format. The master template is `locale/sudoku.pot`.

Supported locales: `en`, `pt_BR`, `es`, `fr`, `de`.

- Scene strings use Godot's AutoTranslate (default ON for Control nodes) — text in `.tscn` files is automatically passed through `tr()`.
- Dynamic strings in GDScript use `tr("KEY")` or `tr_n("KEY_SINGULAR", "KEY_PLURAL", n) % n`.
- Nodes that must NOT be translated (e.g. the locale selector itself) set `auto_translate_mode = 2` in the scene file.
- `GameState` validates the locale from `settings.cfg` against `VALID_LOCALES` before applying it.
- `*.translation` (compiled binary files generated by Godot from the .po sources) are gitignored.

To add a new language: copy `sudoku.pot` → `locale/<code>.po`, fill `msgstr` values, register in `project.godot` under `internationalization/locale/translations`, add to `LOCALES`/`LOCALE_NAMES` in `main_menu.gd` and `VALID_LOCALES` in `game_state.gd`.

## Font

`assets/fonts/Outfit.ttf` is set as the project-wide default font. For bold weight, use `FontVariation`:

```gdscript
var bold := FontVariation.new()
bold.base_font = load("res://assets/fonts/Outfit.ttf")
bold.variation_opentype = {"wght": 900}
bold.variation_embolden = 1.0
label.add_theme_font_override("font", bold)
```

Do not use `SystemFont` — it overrides the project font and ignores Outfit.

## GDScript conventions

- `range()` returns a `Range` in Godot 4 — wrap in `Array()` before `.shuffle()`: `Array(range(81))`
- All styling (colors, borders, fonts) is applied in GDScript via `add_theme_*_override`, not in scene files
- Border widths on cells: `THICK = 3` on box boundaries (every 3rd row/col), `THIN = 1` elsewhere
- `PopupMenu` extends `Window`, not `Control` — it has no `get_minimum_size()`; use `.size` (Vector2i) instead
- Use `visibility_changed` (not `about_to_popup`) to reposition a PopupMenu after Godot sets its position
- OptionButton caret spacing is controlled by the `arrow_margin` theme constant, not `content_margin_right`
- When connecting game.gd to GameState autoload signals, always disconnect in `_exit_tree()` to prevent signal accumulation across play sessions

## Testing

Tests use the **GUT** plugin (`addons/gut`, version 9.6.0). Test files extend `GutTest`. `test_sudoku.gd` covers pure logic; `test_game_state.gd` uses `watch_signals()` / `assert_signal_emitted()` to verify state mutations and signal emissions.
